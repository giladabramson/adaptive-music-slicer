# Adaptive AI Music Engine — Python dependencies
#
# Install (CPU is fine for an MVP; demucs will pull in torch automatically):
#
#   pip install -r requirements.txt
#
# System dependency NOT installable via pip — required for MP3 read/write and
# for librosa to decode compressed input formats:
#
#   ffmpeg   (https://ffmpeg.org/  •  Windows: `winget install Gyan.FFmpeg`)
#
# ffmpeg is REQUIRED: Demucs separates to MP3 (its WAV writer needs the
# optional torchcodec on torch>=2.9) and pydub needs ffmpeg to read
# those MP3 stems back for slicing.

transformers>=4.40     # Step 0 — MusicGen text->music (Hugging Face)
accelerate>=0.26       # faster/leaner MusicGen weight loading
sentencepiece>=0.2     # tokenizer backend for MusicGen's T5 text encoder
demucs>=4.0.0          # Step 1 — source separation (htdemucs 4-stem model)
librosa>=0.10.1        # Step 2 — BPM / beat detection (MIR)
pydub>=0.25.1          # Step 3 — sample-accurate slicing & export
soundfile>=0.12.1      # libsndfile backend used by librosa for WAV/FLAC/etc.
numpy>=1.24            # numeric helpers (librosa dependency, used directly)

# MusicGen's weights (~2 GB for -small) download once from the Hugging
# Face Hub to ~/.cache/huggingface on first --generate use. CPU
# inference is slow (minutes); a GPU is much faster. MusicGen is
# INSTRUMENTAL — it does not generate vocals.
